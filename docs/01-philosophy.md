# 01 · 方法论与边界

## 1. 为什么要先谈这个

本项目要把八字翻译成数学。但"数学化"是个被滥用的词——画几张函数图也能叫数学化。
所以先说清楚：我们在做哪一种数学化，它能量化什么、不能量化什么。

## 2. 科学方法的最小定义

剥到最简，自然科学（物理、化学、生物……）的方法可写成三步：

1. **观察**：发现某种反复出现的规律。
2. **形式化**：发明一套语言（公式、符号、模型）去描述这条规律。
3. **验证**：用这套语言做可被检验的预测，错了就改。

中国古人在"生辰→人生"这件事上，其实**走完了前两步**：

- 观察：出生时刻与气候/太阳周期绑定，而周期与人体发育节律相关——这个直觉并非毫无根据。
- 形式化：他们没有微积分和群论，于是用 **天干地支 + 五行 + 阴阳** 这套符号去编码周期与关系。

这套符号语言**本身是有限的、离散的、有内部结构的**——这正是它能被代数翻译的根本原因。

## 3. 八字体系里"已经像数学"的部分

不需要现代化，传统表述里就藏着大量数学结构：

- 六十甲子是 60 而不是 120 —— 因为有奇偶约束（→ 子群）。
- 相生是一个 5 循环，相克**恰好**是它的平方 —— 生克不是两套规则（→ 循环群 C₅）。
- 八卦是 2³，六十四卦是 2⁶ —— 莱布尼茨一眼看出是二进制（→ F₂ 向量空间）。

这些不是事后硬贴的标签，而是体系**自带**的对称性。把它们写干净，就是本项目的数学内核。

## 4. 数学化能交付什么

| 能做 | 不能做 |
|----|----|
| 让逻辑**精确无歧义** | 让经验预测**自动为真** |
| 让体系**可计算、可程序化** | 证明"八字真能预测性格" |
| **形式化检验内部一致性**（找出自相矛盾的地方） | 取代统计检验 |
| 为经验预测提供**可测试的精确定义** | 给出因果机制 |

一句话：**数学化解决"逻辑对不对"，不解决"灵不灵"。** 但只有逻辑对齐了，"灵不灵"才是一个可回答的问题。

## 5. 本项目的立场

- **不预设结论**：既不是为了"证明八字是真的"，也不是为了"揭穿它是假的"。
- **把争论拉到同一张桌子上**：现在争论八字，常常是双方对"八字到底是什么"理解就不一样，
  等于在比两套不同定义。本项目的产出会让任何讨论都站在同一组精确定义之上。
- **可证伪化接口**：内核形式化后，任何经验断言（"某格局的人更外向"）都可以被翻译成一个
  **明确的、可被数据检验的命题**——这才是把它从"玄学"拉回"可讨论"的关键一步。

## 6. 架构含义

这个边界直接决定了项目架构（见 README 第 3 节）：

- **数学内核**只负责"逻辑对不对"——它必须是**应用无关**的纯代数，不夹带任何经验断言。
- **预测/风水/健康**这些夹带经验断言的部分，都放在**消费者层**，可被替换、可被统计证伪。
- 一旦内核里混进了"木多的人脾气大"这种经验句，内核就不再中立了——这是要严格避免的污染。

所以：**把"是什么"（代数）和"意味着什么"（经验）死死分在两层。**
