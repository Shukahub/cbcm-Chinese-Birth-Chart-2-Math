# 10 · 性格数据驱动验证 · 调研与计划

> 把"八字能否预测性格"变成**可证伪的统计命题**。本文是动手前的调研与方案。
> 呼应 [01-philosophy](01-philosophy.md)：形式化让"灵不灵"从嘴仗变成可回答的问题。

---

## 1. 数据需求（精确字段）

| 字段 | 八字用途 | 验证价值 |
|----|----|----|
| 出生**年** | 年柱、生肖、年五行 | 粗（每年才变） |
| 出生**月** | 月柱、月令 | 中（每月变，但=纯季节信号） |
| 出生**日** | 日柱 | 高 |
| 出生**时**（到 2 小时时辰） | 时柱、全盘 | 高（**最难获取**） |
| 人格量表分（Big Five / OCEAN） | 标签 | 必需 |

> ⚠️ 关键矛盾：**人格数据集**几乎都不收出生日期（只收年龄）；
> **出生时辰数据集**（占星/八字类）又没有经过心理计量验证的人格分。
> 二者交集在公开数据里**基本不存在**——这是本计划的核心约束。

## 2. 数据可得性调研

| 数据源 | 有人格 | 有出生日期 | 有出生时辰 | 可得性 | 评价 |
|----|----|----|----|----|----|
| Open Psychometrics Big Five（~1M） | ✅ OCEAN | ❌ 仅年龄→出生年 | ❌ | 公开下载 | **Phase 1 起点**：可做年柱级粗验 |
| Kaggle "Big Five"（~1M） | ✅ OCEAN | ❌ 仅年龄 | ❌ | 公开 | 同上 |
| Open Psychometrics 星座人格量表 | ✅(自编) | ✅ 出生日期 | ❌ | 公开(部分) | 含生日但量表非学术级，仅参考 |
| SOEP（德国面板，科学使用版） | ✅ BFI-S | ✅ 出生**月+年** | ❌ | 注册可得 | **Phase 2 起点**：月柱级，需季节控制 |
| UKHLS / Understanding Society（英） | ✅ Big Five | ✅ 出生日期 | ❌ | 注册可得 | 同上 |
| Add Health / HRS / PSID（美） | ✅ 部分 | ✅ 出生日期 | ❌ | 受限申请 | 流程长 |
| 英国出生队列 NCDS/BCS70 | ✅ Big Five | 全员**同月同周**生 | ❌ | 申请可得 | **不可用**：出生月零方差 |
| Gauquelin 数据集 | ❌(职业/成就) | ✅ 含时辰 | ✅ | 公开归档 | 历史价值（"火星效应"），非人格 |
| 自采问卷（生辰+Big Five） | ✅ | ✅ | ✅ | 需招募/伦理 | **Phase 3**：唯一能做全盘的路径 |

**结论**：
- 全盘八字（需时辰）的公开数据**基本不存在** → 全盘验证需自采。
- 最现实的公开数据起点是 **Open Psychometrics Big Five**（出生年由年龄反推），
  只能做**年柱/生肖/年五行**级的最粗验证。
- 进一步需要 **SOEP / UKHLS** 这类面板（注册可得），做到**月柱**级，但必须处理季节混淆（§4）。

## 3. 核心挑战

### 3.1 出生时辰缺失
公开人格数据无人收时辰 → 全盘八字（四柱+藏干十神）无法排。
务实策略：**按可得字段降级**——年→年月→年月日时，分三阶段，能验多少验多少。

### 3.2 季节混淆（最致命，必须处理）
出生**月**既决定八字月支（春生=木旺），又**已知**独立影响心理/发展变量
（相对入学年龄、孕期营养、早期疾病暴露、光照……）。
所以"出生月↔人格"的任何相关，几乎都是**季节效应**而非"八字逻辑"。

> 这意味着：月柱级验证若不做混淆控制，测到的只是"出生季节影响人格"——
> 与八字是否成立**无关**。科学的做法是**控制出生月**后，看八字框架是否有**增量**解释力。

## 4. 三阶段方案

### Phase 1 · 年柱级（公开数据，立刻可做）
- **数据**：Open Psychometrics Big Five（`age`→出生年；`country`/`gender` 作协变量）。
- **特征**：年柱干支、生肖、年五行（5 类）、纳音年行。
- **假设**（预注册示例）：年五行分组在 OCEAN 任一维度的均值差为 0。
- **做法**：ANOVA / 回归，控制年龄、性别、国家；多重比较校正。
- **产出**：可复现 notebook + 结论（很可能零/微弱效应）。
- **局限**：年柱每年才变，统计效力低；且年五行≠真正"八字"。

### Phase 2 · 月柱级（注册面板数据，需申请）
- **数据**：SOEP 或 UKHLS 科学使用版（出生月+年 + BFI/Big Five）。
- **特征**：年柱 + 月支/月令（→ 日主旺弱的月令支撑）。
- **关键**：回归中**加入出生月固定效应**，吸收全部季节混淆；
  再看八字特征是否有**增量**预测力（ΔR² / 似然比检验）。
- **基线对比**：模型 A=仅出生月；模型 B=出生月+八字特征。B 显著优于 A 才算信号。
- **产出**：混淆控制下的诚实检验报告。

### Phase 3 · 全盘（自采数据，重投入）
- 招募被试填：出生年月日**时** + Big Five（+ 可选自评/他评）。
- 特征：四柱、藏干十神、旺弱、喜用（用 core 全量产出）。
- 同样须控制出生月/季节。
- **现实**：需伦理审查、招募、样本量（数百~数千），是独立子项目。

## 5. 统计协议（科学诚实清单）

1. **预注册假设**：动手前把传统断法翻成有限条命题（如"日主偏旺→外向""木旺→开放性"），
   写进文档冻结，杜绝事后挑结果。
2. **多重比较校正**：5 行 × 5 特质 = 25 组合，加生肖等 → 用 **Benjamini-Hochberg (FDR)** 或 Bonferroni。
3. **混淆控制**：出生月固定效应 + 人口学协变量（性别、年龄、国别）。
4. **基线对比**：八字模型 vs 仅出生月模型 vs 随机；报告 **增量** 效应量（ΔR²、Cohen's d）。
5. **效力分析**（a priori）：事先算需要多大样本才能检出小效应。
6. **报告阴性**：零结果照实发表——这正是本项目"中立"姿态的体现。

## 6. 预期结果（诚实）

- Phase 1 大概率**零/微弱**：年柱太粗。
- Phase 2 控制季节后**很可能归零**：现有"出生月↔人格"文献效应本就很小且不稳。
- 若真有残留信号，那会是**有趣的发现**（值得追，但需独立复制）。
- 任何阶段都不追求"证明八字为真"——交付的是**可复现的方法与诚实的结论**。

## 7. 工程产出（管线结构）

复用 `cbcm-core`，新建 `packages/app-validation/`：

```
app-validation/
  src/cbcm_validation/
    features.py    # chart → 特征向量（日主旺弱/五行强度/十神分布/生肖…）
    data.py        # 加载 Open Psychometrics / SOEP（按 phase 适配）
    stats.py       # 回归 + 季节固定效应 + FDR + 基线对比
    report.py      # 生成结论报告（效应量、置信区间、是否预注册支持）
  tests/
  notebooks/       # 分析 notebook（Phase 1 起）
```

特征层 `features.py` 几乎是对 core 的纯映射，**不引入新断言**；
统计层 `stats.py` 把"灵不灵"量化。整个包仍是"消费者"，不污染内核。

## 8. 推进建议

1. **先做 Phase 1**：下 Open Psychometrics Big Five，打通 `features→stats→report` 管线，
   把季节控制/多重比较框架跑通（即便年柱很粗，框架可复用到 Phase 2/3）。
2. **平行申请 SOEP/UKHLS**（注册流程数周），为 Phase 2 铺路。
3. Phase 3 视兴趣与资源再立项。

> 一句话：**数据决定能验多深，方法决定是否可信。** 先把方法论框架（混淆控制+预注册+基线对比）
> 在 Phase 1 跑顺，是性价比最高的第一步。
