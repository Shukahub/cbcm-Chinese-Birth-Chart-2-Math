# ChineseBirthChart2Math

把中国传统的 **生辰八字 / 天干地支 / 五行 / 易经** 体系，翻译成一套**严格的代数（数学）语言**的探索项目。

---

## 1. 动机

很多人把生辰八字归为"玄学 / 不科学"。但如果回到科学方法的最小定义——
**观察现象 → 用形式语言总结规律 → 用该语言验证**——那么中国古人其实已经完成了前两步：
他们没有现代数学工具，于是用自己的符号语言（甲乙丙丁、子丑寅卯、金木水火土）去总结
太阳运行周期、节气气候、进而推断出生时刻与人的情志/身体之间的关联。

本项目的核心猜想是：**这套符号体系并非随机，而是一套有限、离散、自洽的代数结构。**
它完全可以用群、环、置换、向量空间等现代数学语言逐字翻译过来。翻译之后：

- 逻辑变得**精确、无歧义、可计算**；
- 内部一致性可以**形式化检验**；
- 为后续**统计检验**经验预测留出干净接口。

---

## 2. 性质与边界

- **性质**：玩乐 / 探索 / 学习。不宣称"证明玄学为真"。
- **数学化能做的**：把*逻辑结构*精确化、可计算化、可验证内部一致性。
- **数学化不能做的**：让*经验预测*（如"八字能预测性格"）自动变真。这一步必须靠统计检验，
  而统计检验的前提恰恰是逻辑无歧义——这正是本项目要交付的底层。
- 换句话说：本项目的产出是一个**中性的形式化内核**，它既不偏袒"信"也不偏袒"不信"，
  而是把争论从"你说你的、我说我的"拉回到"在同一套精确定义上检验"。

## 3. 架构原则

```
                  ┌─────────────────────────────────────┐
   应用层(消费者)  │  预测性格情志  │  健康  │  风水  │  择日 ... │   ← 可替换、可被统计检验
                  └─────────────────────────────────────┘
                                   ↑ 都调用同一套
                  ┌─────────────────────────────────────┐
   算子层         │  十神 · 刑冲合害 · 神煞 · 纳音 ...   │   ← 状态→标注 的映射
                  └─────────────────────────────────────┘
                                   ↑
                  ┌─────────────────────────────────────┐
   数学内核       │  干支 Z₆₀ · 五行 C₅ · 易经 (F₂)ⁿ     │   ← 应用无关，纯代数
   (本项目重心)   │  八字状态空间 G⁴ + 排盘              │
                  └─────────────────────────────────────┘
```

**关键原则：数学内核应用无关。** 预测、风水、健康等都只是读同一个内核的不同消费者。
内核稳定后，加一个应用分支 = 套一层映射，不碰底层。这与"后期应用分支"的规划一致。

## 6. 文档体例

理论文档（03–06）统一采用**"先古法、后翻译"**结构：

1. **§0 传统说法（古法原典）**：用古人自己的话、带出处（《洪范》《系辞》及口诀）把概念讲清楚；
2. **数学翻译**：逐条把古法译成代数对象；
3. **小结**：即"古法 ↔ 代数"对照表。

这么做是因为本项目的本质是**翻译**——古法为源语言、代数为目标语言。
让懂八字不懂群论、或懂数学不懂八字的人，都能看到两边逐条对应。
